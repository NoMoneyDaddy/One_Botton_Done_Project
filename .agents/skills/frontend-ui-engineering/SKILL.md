# frontend-ui-engineering

做能上線的 UI，不做 AI 套版味。

## 讀取順序

1. 讀 `.agents/skills/ui-ux-designer/SKILL.md`
2. 讀 `.agents/skills/website-best-practices/SKILL.md`
3. 讀 `.agents/skills/responsive-design-best-practices/SKILL.md`

## 必做

- 先定主要資訊層級
- 用語意化 HTML
- 確保鍵盤可操作與基本無障礙
- 做桌機與手機都成立的 layout
- 避免預設、平庸、可替換感太強的介面

## 驗證

- 看真實畫面，不只看 code
- 有瀏覽器工具就檢查：
  - console
  - DOM
  - network
  - layout overflow

## 規則

- 沿用既有 design system
- 沒有 design system，就先建立最小設計基線
- 視覺決策要能說明，不要亂堆效果
