# ui-ux-designer

上游靈感來自 `sickn33/antigravity-awesome-skills` 的 `ui-ux-designer`。

## 核心原則

- 先釐清目標使用者、裝置、場景
- 優先做可用、可讀、可達的介面
- 系統化優於一次性美化
- 保持與既有設計語言一致；若無既有語言，再建立明確視覺方向

## 必查清單

- 資訊層級是否明確
- 互動是否有狀態回饋
- 主要 CTA 是否明顯
- 錯誤、空狀態、loading 是否完整
- 色彩對比、鍵盤操作、語意結構是否到位
- 手機版是否可操作，觸控目標是否足夠

## 與其他 skill 的搭配

- 視覺與 layout：搭配 `responsive-design-best-practices`
- 前端落地：搭配 `website-best-practices`
- React/Next：再搭配系統或官方的 React best practices
